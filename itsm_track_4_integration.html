🎯 Track Goal: Enable the team to layer Agents on top of existing customer workflows and products — creating an AI orchestration layer without needing to "rip and replace" a customer's existing tech stack. Demonstrated by hands-on exercises in demo environments.
💡 Core positioning: Salesforce Agentforce ITSM is not a replacement mandate — it's an AI orchestration layer that integrates natively with ServiceNow, Jira, Freshservice, Workday, and legacy ITSM tools. This is the primary competitive differentiator vs. rip-and-replace alternatives.
⚠️ Current Functional Gaps — verify before scoping any integration engagement. Known co-existence and platform limitations as of June 2026.

ServiceNow Co-Existence — No OOTB Support

Not supported OOTB without custom development. Positioning Agentforce as the AI layer on top of ServiceNow requires custom MuleSoft or Platform Events integration — no native bi-directional sync. Scope custom build time explicitly.

Custom Dev Required

Jira / Azure DevOps — No OOTB Co-Existence

No native co-existence support. Integrating Agentforce with Jira Service Management or Azure DevOps requires custom API-led connectivity via MuleSoft. Do not promise OOTB sync in scoping.

Custom Dev Required

CMDB Regional Availability

US and Germany only through September 2026. CMDB discovery and CI topology are not yet available in other regions. Flag in any EMEA (outside Germany) or APAC engagement.

Regional Limitation

VPN / Off-Network Device Detection

Service Graph Connector cannot discover devices off-VPN. Endpoint agents require devices to be on the corporate network or VPN for discovery. Off-VPN remote workers represent a CMDB coverage gap.

🖥️ IT Service — Co-Existence & Orchestration Strategy
How does CMDB solve API integration, privacy, security, and user context pain points for IT teams?IT ServicePJF Notebook

API Integration

Agentforce IT Service offers 100+ pre-built connectors (Jamf, Workday, Google, Microsoft) out of the box. Because the CMDB operates on a deeply unified platform, IT teams can seamlessly connect their existing IT landscape without building complex API integrations from scratch or relying on "bolt-on" third-party solutions.

Privacy and Security

The CMDB runs on Data Cloud with a "zero-data copy" architecture alongside Retrieval-Augmented Generation (RAG). AI agents securely search and reason over enterprise data without duplicating or moving it. The Salesforce platform provides built-in security, trust, and strict AI guardrails.

User Context and Visibility

  • The Service Graph: Maps complex relationships and dependencies between hardware, software, and applications — a real-time "contextual map" for both human fulfillers and AI agents to immediately see how an outage impacts a specific user, assess change risk, or automatically perform root-cause analysis.
  • Unified Employee Profile: Asset data from the CMDB (employee's assigned laptop or software) is tied to a Unified Employee Profile — a 360-degree view showing active IT incidents, HR data, and assigned hardware all in one place.
How can ITSM be integrated into a build-operate-transfer model?IT ServicePJF Notebook

The notebook sources do not contain explicit guidance on integrating the ITSM solution into a Build-Operate-Transfer (BOT) model. The sources primarily focus on standard ITIL processes, deploying the platform using Salesforce Go, and utilizing AI agents for ongoing IT operations.

However, the Agent-Driven Ramp Plan in the main Learning Journey provides a ProServ-specific framework that maps to BOT phases:

  • Build (Days 1–60): Tiger Team intensive training, reference architecture development, solution bundles, and first POC SOWs
  • Operate (Days 60–90): Market entry, governance & PM rigor, onboard PM/TA/SA, create and reuse agents and accelerators
  • Transfer (Day 90–120+): Global delivery model, establish feedback loop, Primary Consultant becomes team lead, SAs create accelerators
What framework can ITSM be packaged and distributed in?IT ServicePJF Notebook

1. Data Kits for Data Components

A Data Kit packages Data Cloud components — data model objects and data streams — so they can be readily deployed into an org. The predefined Agentforce IT Service Data Kit includes bundles of data streams (IT Service Core and Analytics bundles) and search indexes.

2. Salesforce Go "Solution Bundles" for App Features

  • IT Service Standard Solution: Incident, Problem, Change, Release Management, Self-Service Portal, SLA Management
  • IT Service Enhanced Solution: Everything in Standard, plus CMDB and Agentforce capabilities
  • IT Service Sample Data Solution: Packages sample data to instantly populate the org for exploration

3. ITIL (Conceptual Framework)

The platform's workflows are packaged and distributed according to the ITIL framework — the industry-standard best practices for high-quality and efficient IT service management.

👥 HR Service — Integration Patterns
How does IT Service integrate with HR needs on a shared platform?IT + HR UnifiedIT Service Notebook
  • Combined AI Agents for Employees: The AI Agents for Employees license provides both "IT and HR service" across the Employee Portal, Slack, Microsoft Teams, and Webchat — a single conversational agent routes requests to the right backend, whether IT or HR.
  • Unified Data Model: IT and HR Services use a Unified Data Model combining IT, HR, Finance, and employee records. A single employee profile displays IT incidents (Wi-Fi issues, software access) alongside HR cases (payroll errors, badge issues) and HR data (title, location).
  • Shared and Secure Environment: IT and HR share a dedicated employee org, entirely separate from the external Customer Service CRM org. This protects confidential employee data (HIPAA, payroll, internal IT infrastructure) and ensures separate release cycles and security controls.
  • Seamless Contextual Handoffs: Because IT and Customer Service operate on the same data model, both teams share a complete real-time view of any issue — no need for users to repeat themselves or toggle between different systems.
⚠️ Single-Org Limitation: IT Service and HR Service currently require separate employee orgs. Single-org design is roadmap only — separate employee org required in all current engagements.
