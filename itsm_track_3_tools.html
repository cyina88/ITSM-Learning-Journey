AI Agent Capabilities — Tool-Level Deep Dives

Q&A from the IT Service notebooks on specific Agentforce tools and how they work.

🖥️ IT Service — Employee-Facing AI Agents
How do AI agents autonomously resolve issues for employees?IT ServiceIT Service Notebook

AI agents provide proactive, conversational support directly in Slack, Microsoft Teams, and the Employee Portal.

  • Generative Knowledge Answers: Agents scan the knowledge base and integrated enterprise data. For a VPN issue in Slack, the agent uses "Answer Questions with Knowledge" to instantly surface and summarize the troubleshooting steps needed.
  • Automated Service Requests: Handles routine requests — password resets, account unlocks, software provisioning, adding users to distribution lists — without human IT intervention. The agent confirms details, submits the request, and routes approval to the manager.
  • Proactive Endpoint Remediation: Endpoint agents integrate with device monitoring tools to proactively detect issues before the employee reports them. If a computer has high memory usage, the agent pings the employee in Slack, identifies the problem, and asks permission to remotely restart the app.
  • Automated Escalation and Ticketing: If too complex for auto-resolution, the agent automatically creates a detailed incident ticket with full context from the natural-language conversation and routes to the IT service desk. Employees don't need to repeat themselves or fill out forms.
What is the role of endpoint agents in resolving issues?IT ServiceIT Service Notebook
  • Proactive Detection via Integrations: Endpoint agents integrate directly with Jamf, Intune, Kandji, Workspace ONE, and CrowdStrike. They receive real-time alerts when something goes wrong on a remote device — tracking metrics like abnormal CPU spikes or high memory usage.
  • Conversational Remediation: When an issue is detected (e.g., high memory causing device slowdowns), the agent immediately starts a conversation with the employee in Slack, notifying them of the specific problem and asking permission to execute a fix.
  • Automated Remote Actions: Once the user grants approval, the endpoint agent autonomously kills and restarts a resource-heavy application (e.g., Google Chrome), then confirms memory levels have returned to normal.
  • Broad Endpoint Coverage: Extends beyond laptops and desktops — endpoint agents peer into Kubernetes containers and Virtual Desktop Infrastructure (VDI) environments such as VMware, Citrix, and hypervisors.

Especially valuable for non-technical staff — hospital nurses and similar roles — who need seamless background IT support without manual troubleshooting or ticket submission.

🖥️ IT Service — Fulfiller-Facing AI & Proactive Assistance
What is the Proactive Assistance tool in the IT Service Console?IT ServiceIT Service Notebook

The Proactive Assistance tool is a widget in the IT Service Console that consolidates several Agentforce actions — providing a 360-degree, AI-driven view of an incident. It saves fulfillers hours of manual research and troubleshooting.

Core Capabilities

  • Check Attributes: Validates key incident fields and can automatically update them based on historical context (e.g., changing impact and urgency from medium to high).
  • Incident Summary: High-level overview of the incident including relevant details, SLA statuses, and associated knowledge articles or tasks.
  • Root Cause Summary: Analyzes previous similar incidents to highlight contributing factors and provides a high-level summary of what might be causing the current issue.
  • Associate Similar Incidents: Automatically surfaces and links similar past incidents — gives the fulfiller broader context and helps evaluate whether escalation to a Major Incident is needed.
  • Propose Resolution Summary: Reviews how similar issues were solved in the past, alongside relevant knowledge articles, to provide clear step-by-step recommended actions.
  • Agent Based Assignment: Einstein-powered action that automatically suggests the right person, team, or queue to assign the incident to — analyzing subject, description, priority, and past resolution skills.

Fulfillers can expand and view AI-generated insights directly on the incident record. If new data comes in, they can "refresh" to regenerate results in seconds. Also being extended to Problem Management (Proactive Assistant Problem, added in the 260 release).

What is the Propose Resolution Summary action?IT ServiceIT Service Notebook

The Propose Resolution Summary action is an Agentforce capability available within the Proactive Assistance tool for both incidents and problems. It does the heavy lifting of troubleshooting by instantly digesting how similar issues were handled in the past.

The AI evaluates existing knowledge base articles alongside resolution details of past related incidents or problems, then synthesizes historical data into clear, step-by-step recommended actions tailored to the current issue.

Example: For a VPN connection failure, the action might suggest verifying the portal address based on a specific knowledge article, then testing for firewall interference based on how a previous similar incident was successfully resolved.

What is the Create Incident Root Cause Summary action?IT ServiceIT Service Notebook

The Create Incident Root Cause Summary (also called Incident Cause Analysis) is an Agentforce AI capability that automatically generates a proposed root cause summary for an active incident by analyzing:

  • Current and Past Incident Data: Details and descriptions of the active incident compared against similar historical incidents to identify patterns and contributing factors.
  • Affected Configuration Items (CIs): The agent examines asset dependencies mapped in the CMDB and Service Graph. E.g., traces a failing VPN service node back to a change request executed on a connected gateway cluster.
  • Knowledge Articles and Related Records: Digests existing knowledge base articles, known errors, and open problem records for grounded context.

Accessible via the Proactive Assistance tool in the Incident Management app. Saves significant investigation time and accelerates resolution.

How does AI use historical data to propose resolution steps?IT ServiceIT Service Notebook
  • Searching Past Resolutions: The AI scans for previously resolved incidents or problems that share characteristics with the current issue.
  • Extracting Proven Steps: Identifies related historical records and digests the specific resolution summaries, action plans, and tasks that were used to fix those past issues. E.g., finds that a past VPN ticket was resolved by reviewing recent software changes and rolling back a specific implementation.
  • Synthesizing with Knowledge Base Data: Cross-references historical actions with relevant troubleshooting steps in the knowledge base articles.
  • Generating Actionable Recommendations: Synthesizes combined historical context into a clear, step-by-step list of "Recommended Actions" — allows fulfillers to immediately apply proven fixes without manually hunting through old tickets.
How does AI prioritize which incidents to analyze first?IT ServiceIT Service Notebook

The AI helps prioritize incidents for human IT teams through several mechanisms:

  • Predicting SLA Breaches: Uses predictive models and historical data to anticipate which active incidents are likely to breach their SLAs. Proactively highlights at-risk issues so fulfillers know to prioritize them.
  • Predicting Major Incidents: Evaluates patterns in incoming tickets — e.g., a sudden surge of VPN connectivity issues within the last hour — to predict the likelihood of escalation to a Major Incident.
  • Prioritization and Categorization Checks: For individual tickets, the AI actively checks if the incident has been assigned the correct priority, category, and subcategory, offering intelligent correction suggestions.
  • Rule-Based Automation: Alongside AI-driven insights, the IT Service Console uses automated, rule-based priority settings to consistently categorize and prioritize the incident backlog.
🖥️ IT Service — Knowledge Management
How do AI agents use Knowledge for troubleshooting?IT ServiceIT Service Notebook
  • Generative AI-Powered Answers for Self-Service: Instead of forcing employees to hunt through long articles, agents scan the centralized knowledge base and integrated enterprise data to instantly surface and summarize troubleshooting steps — conversationally in Slack.
  • Proposing Resolution Steps for IT Fulfillers: The AI runs a "Propose Resolution Summary" action — digesting relevant knowledge articles alongside past resolved incidents to provide the human fulfiller with clear, recommended step-by-step troubleshooting actions.
  • Intelligent Knowledge Recommendations: During incident or problem diagnosis, agents actively suggest relevant knowledge articles, known errors, and temporary workarounds based on historical data and the specific context of the issue.
  • Automated Knowledge Creation: After resolving a new or complex issue, the AI automatically drafts a new knowledge article detailing the root cause, steps taken to fix it, and preventative best practices. Continuous learning loop ensures future troubleshooting is significantly faster.
  • Smart, Permission-Aware Search: AI-powered search consolidates data from multiple sources — including SharePoint, Zendesk, or Google Drive — into a single summarized answer, ensuring employees and agents only access information they're authorized to view.
What is the Knowledge Creation action and how does it work?IT ServiceIT Service Notebook

The Knowledge Creation action is an Einstein-powered Generative AI capability that automatically drafts new knowledge base articles when an IT team resolves an issue.

How It Works

  • Contextual Extraction: Pulls relevant data directly from IT records — the incident's identified root cause and specific steps taken to reach resolution.
  • Generative Drafting: Autonomously drafts a comprehensive new knowledge article. E.g., if a service outage was resolved by reverting a faulty security patch, the article details the root cause, resolution applied, and preventative best practices for testing future deployments.
  • Continuous Improvement: In addition to creating new articles, the AI evaluates historical resolutions to suggest updates to existing articles, or drafts new "known error" records and temporary workarounds.
  • Human Review & Publishing: The IT team reviews the AI-assisted draft before publishing. Once published, the knowledge base immediately empowers faster employee self-service.

Availability

Included natively in the Agentforce IT Service Desk Unlimited Edition (UE), or accessible through the Agentforce for Service (A4S) add-on.

🖥️ IT Service — Enterprise Knowledge & Experience
What is Enterprise Knowledge and how does it work?IT Service262 Notes

Enterprise Knowledge enables ingestion from external sources — Google Drive, Confluence, SharePoint — into Data Cloud to create a unified Salesforce knowledge base.

  • Harmonization Layer: Formats external content for readability before surfacing in Salesforce.
  • Hybrid Search: Suggests relevant articles based on incident subject and description context.
  • RAG-Style Direction: Current keyword search is insufficient — roadmap push toward full RAG-style search for Enterprise Knowledge.
⚠️ Not Yet Available: AI-generated summaries within Enterprise Knowledge are on the roadmap but not yet delivered. Configuration must be performed explicitly — not automated as part of first-run experience.
What changes came in 262 for Employee & IT Agent experience?IT Service262 Notes
  • Email to Incident/Case/Service Request: Email threading and fulfiller response capabilities introduced in 262; additional refinements targeted for 264.
  • Slack App Consolidation: Moving toward a unified "Salesforce app" for Slack to handle notifications and approvals — reduces need for multiple app installations. Better adaptive card UX required. Do not present current non-final iterations to customers.
  • Microsoft Teams: Automated setup improvements for the IT service app. Internal troubleshooting "cloud agent" available for GTM/SCE teams to debug setup issues.
  • Change Advisory Board (CAB): Integrates governance into platform but deprioritized for this release — lacks immediate customer requests.
What is the current state of Agentforce agent design?IT Service262 Notes

The 262 enablement sessions surfaced significant concerns about the current agent design approach:

  • "One-Shot" Agent Problem: Current reliance on one-shot agents and service catalog-based workflows is characterized as "brittle" and "backward-looking." Team is prioritizing an agent script approach that is more legible and extensible.
  • Multi-Agent Orchestration: Available for Microsoft Teams and portals; Slack remains a work in progress (P-stage) — significant gap for upcoming releases.
  • Demo Quality Bar: Current device security assistance agent demo is not ready for customer presentations. Mandate: build a new production-ready demo focused on a single high-quality agent (e.g., software access management) rather than many shallow agents.
  • Non-Catalog Logic Needed: Must move beyond service catalog-only approach — need agent skills, tool calling APIs, and proper workflow design.
  • Parallel Execution Gap: Current multi-agent path is serial — parallel execution needed for efficiency.
  • Human Escalation: Live agent handoff must work for both single agents and multi-agent orchestrated flows across all platforms.
What are the confirmed roadmap investments from 262 feedback?IT Service262 Notes
  • Unified Fulfiller View: Investment planned for a unified ticket view for fulfillers to reduce tool-switching.
  • Knowledge Agent + Configurable Article Suggestions: Planned for future releases to improve self-service accuracy.
  • MS Teams & Slack Enhancements: "My Assets" visibility in collaboration tools + richer Agentforce interactions prioritized.
  • Multilingual Support: Multilingual template management within the notification framework — mandatory as product scales.
💲 Licensing — UEL vs. CCP
What is UEL and how does it differ from CCP?IT + HR Unified262 Notes

UEL (Universal Employee License) — also called EUL (Employee User License) — is the platform-based license that now underpins the AUL and AI Agents for Employees SKUs. CCP (Customer Community Plus) was the original license at launch and has been replaced by UEL for all new deals.

What is the recommendation for existing CCP customers?IT + HR Unified262 Notes

Strong recommendation is to migrate CCP customers to UEL as soon as possible. CCP-based licenses will be supported until EOL (Jan 31, 2031) but no new OOTB agents, workflows, or connectors will be added to CCP.

  • Migration Tool: A Summer 2026 migration tool is being released to support the CCP → UEL transition.
  • Recommended SI for migrations: CustomerTimes.
  • Demo orgs: May currently show CCP-based licenses — expected for now. New users in demo orgs should be set up with UEL to test latest functionality.
