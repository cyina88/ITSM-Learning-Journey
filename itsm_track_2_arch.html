🖥️ IT Service — Data Architecture & CMDB
How does Salesforce process and use data in Agentforce for ITSM?IT ServicePJF Notebook

1. Data Foundation & Ingestion (Data Cloud & Zero Copy)

Agentforce ITSM operates natively on the Salesforce platform using Data Cloud to break down data silos. It relies on a "zero copy" architecture and Retrieval-Augmented Generation (RAG) to ensure AI agents have secure, real-time access to all organizational data without duplicating it.

  • Data Streams and DMOs: Core ITSM data (incidents, problems, change requests) is ingested into Data Cloud via predefined data streams, mapping Salesforce objects to Data Model Objects (DMOs).
  • Search Indexing: Search Indexes are built by "chunking" key fields like incident subjects, descriptions, and categories to enable fast AI querying.
  • Knowledge Integration: External and internal knowledge articles are indexed, providing the foundational grounding data that Agentforce uses to generate accurate responses.

2. Asset Data Processing (CMDB & Discovery)

The CMDB acts as the single source of truth for IT infrastructure, populating Data Cloud with real-time asset configurations.

  • Discovery Scanning: Agent-based and agentless tools scan networks using WMI, SSH, SNMP to detect hardware, software, and cloud resources, sending raw data to the CMDB.
  • Identification Rules & Process Jobs: Identification Rules match incoming discovery data against existing CIs to prevent duplicates. Process Jobs then analyze data to automatically build a Service Graph, mapping dependencies and relationships between assets.

3. AI Reasoning and Action Loops

The AI evaluates requests, selects the appropriate "Topic" (Service Catalog rules or ITIL processes), and executes tasks using connected tools, APIs, and workflows.

4. Practical Uses of ITSM Data

  • Intelligent Incident Triage & Routing: Checks new incidents against historical data to validate attributes, then routes to the most qualified IT fulfiller based on past success.
  • Semantic Association of Records: Semantic searches find and automatically link similar incidents (e.g., identifying 22 VPN failures in 48 hours).
  • Root Cause Analysis & Problem Management: Processes CMDB dependency maps and analyzes related incidents or change requests to generate root-cause summaries and automatically draft Problem records.
  • Automated Knowledge Creation: Einstein uses prompt templates grounded in ITSM record data to transform incidents into reusable Known Error or Troubleshooting Knowledge Articles.
  • Analytics and Leadership Insights: IT Service Analytics bundles bring data into Tableau-powered dashboards for KPI calculation, SLA breach monitoring, asset health, and agent productivity evaluation.
How does a CMDB differ from standard asset management?IT ServiceIT Service Notebook

Asset Management (ITAM)

Focuses on the financial, contractual, and complete lifecycle of IT components.

  • Purpose: Manages the complete lifecycle — procurement, assignment, deployment, maintenance, retirement.
  • Key Focus: Financial tracking, demonstrating ownership, cost optimization.

CMDB

Focuses on operational data, technical configurations, and how components relate to one another.

  • Purpose: Dynamic repository storing Configuration Items (CIs) — functional components of the entire IT infrastructure — along with their specific attributes and dependencies.
  • Key Focus: Support service delivery and processes like incident and change management by providing a "service map" of how components interact and rely on one another.

In short: an asset is tracked for its financial value and ownership; a CI is tracked for its operational role and technical relationships. In a robust IT service system, physical and digital assets are linked directly to their corresponding operational CIs in the CMDB.

How does CMDB solve API integration, privacy, security, and user context pain points?IT ServicePJF Notebook

API Integration

Agentforce IT Service offers 100+ pre-built connectors (Jamf, Workday, Google, Microsoft) out of the box. Because the CMDB operates on a deeply unified platform, IT teams connect their existing IT landscape without building complex API integrations or relying on bolt-on third-party solutions.

Privacy and Security

The CMDB runs on Data Cloud with a "zero-data copy" architecture alongside RAG. AI agents securely search and reason over enterprise data without duplicating or moving it. The Salesforce platform provides built-in security, trust, and strict AI guardrails.

User Context and Visibility

  • The Service Graph: Maps complex relationships and dependencies between hardware, software, and applications — a real-time "contextual map" that both human fulfillers and AI agents use to see how an outage impacts a specific user, assess change risk, or perform root-cause analysis automatically.
  • Unified Employee Profile: Asset data from the CMDB (employee's assigned laptop or software) is tied to a Unified Employee Profile — a 360-degree view showing active IT incidents, HR data, and assigned hardware all in one place.
🖥️ IT Service — Service Graph & Impact Analysis
How does a Service Graph visualize asset dependencies?IT ServiceIT Service Notebook

A Service Graph (also called a Service Map) acts as a visual representation — like a giant map — displaying how different Configuration Items (CIs) are connected. It automatically maps the dependency chain from bottom-level assets (servers, databases, storage) up to the critical business applications and portals they support.

  • Real-Time Impact Analysis: Maps real-time dependencies so IT teams immediately see the "ripple effect" of an outage. If a server goes down, the map visually highlights dependent business services.
  • Contextualizing Active Issues: Surfaces active incidents, problems, and change requests logged directly against specific nodes — not just the assets and connections.
  • Visual Root Cause Identification: Enables fulfillers to trace the origin of an issue visually. E.g., view a failing VPN service node and immediately see that a recent change was executed on a connected gateway cluster.
  • Interactive Drill-Down: Click and drill down into any node to view deeper details, attributes, and specific issues tied to that CI.
How do Service Graphs help identify root causes?IT ServiceIT Service Notebook
  • Visualizing the Entire Dependency Chain: Automatically maps connections between CIs — from an end-user portal down to web servers, application servers, databases, and underlying storage. Teams trace the origin of an issue rather than just treating the symptom.
  • Overlaying Active IT Records: Overlays active incidents, open problems, and recently closed change requests directly onto the affected nodes in the map.
  • Isolating Single Points of Failure: A Problem Manager traces visual dependencies to pinpoint a single failing component as root cause — applies a permanent fix to prevent future outages.
  • Validating AI-Suggested Root Causes: If Agentforce suggests a potential root cause, fulfillers can open the Service Map to visually confirm it.
  • Interactive Drill-Downs: Click specific flagged nodes to drill down into details without toggling between different systems.
What are agentic actions in the Service Graph?IT ServiceIT Service Notebook

The Service Graph integrates with Agentforce to support specific agentic actions that help IT fulfillers analyze and remediate issues autonomously:

  • Advanced Dependency Analysis: AI agents reason over the Service Graph using natural language. Fulfillers can ask "What is the impact if this server goes down?" and the agent analyzes mapped dependencies to explain downstream business impact.
  • Affected CI Suggestion: Instead of manually hunting through the service map, the AI analyzes incident context alongside the Service Graph to provide real-time, intelligent recommendations for which specific CIs are affected.
  • In-Map AI Agent Actions: The Service Map incorporates AI Agent actions — IT teams can execute tasks, view AI-generated summaries, and interact with CI dependencies directly from the visual map.
  • Automated CMDB Health Actions: Agents autonomously perform background actions including Discovery Scan Summarization (analyzing scan logs to diagnose failures), Duplicate CI Detection, and Unhealthy CI Reporting.

Agentic actions transform the Service Graph from a passive visual diagram into an interactive, AI-driven diagnostic tool where agents proactively interpret relationships, suggest affected assets, and help execute remediation tasks.

How does AI analyze dependencies for impact analysis?IT ServiceIT Service Notebook
  • Advanced Dependency Analysis via Natural Language: Ask "What is the impact if this server goes down?" — the AI reasons over the Service Graph to immediately explain downstream business impact.
  • Summarizing the Scope of Outages: During an active incident, AI summarizes the complex web of dependencies, assessing the full scope of the outage's impact — which critical business services, employees, and customers are affected.
  • Automated Risk Assessment for Change Management: Before a system update is approved, the AI uses mapped dependencies to provide automated risk calculations. Evaluates dependent applications connected to the target asset, anticipates potential risks, and helps schedule changes for lowest-impact time windows.
  • Predicting Affected Configuration Items (CIs): As a fulfiller describes an issue or a ticket is logged, the AI provides real-time, predictive suggestions for which specific CIs are affected — immediately linking the ticket to the correct dependency chain.
How does AI predict affected CIs?IT ServiceIT Service Notebook

AI predicts affected Configuration Items through a capability called Affected CI Suggestion:

  • Real-Time Context Analysis: As an IT fulfiller describes an issue or incident details are captured, the AI actively analyzes the context of the situation.
  • Leveraging the Service Graph: The AI grounds its predictions by combining specific incident information with the dependency mapping found in your Service Graph.
  • Proactive Recommendations: Instead of requiring a human agent to manually hunt through the CMDB to link an asset, the AI provides real-time, intelligent suggestions for the exact CIs that are impacted and should be associated with the ticket.

By automating this asset association, the AI improves data accuracy and significantly reduces time to resolution.

👥 IT + HR Unified — Data Model & Governance
What are the benefits of a Unified Data Model?IT + HR UnifiedIT Service Notebook

A Unified Data Model brings IT, HR, Finance, Facilities, and customer records together onto a single foundation — moving from a fragmented "ticketing-centric" model to a unified "customer-centric" data model.

  • Eliminates Silos and Enables Cross-Department Visibility: Breaks down traditional walls between departments. Provides CIOs and leadership with company-wide, real-time context for better decision-making and reporting.
  • Reduces Integration Complexity and Costs: Severely reduces data duplication, integration risks, vendor sprawl, and the high maintenance overhead of connecting multiple disparate legacy systems.
  • Streamlined "Single Pane of Glass" Employee Experience: Employees get a unified profile and a true one-stop-shop for all internal needs — Wi-Fi outage (IT), payroll error (HR), or parking token request (Facilities) all in one seamless interface.
  • Seamless, Contextual Handoffs: Because IT and Customer Service operate on the same data model, both teams share a complete real-time view of any issue. No need for users to repeat explanations or toggle between different systems.
  • Powers Smarter, Cross-Domain AI: When AI agents are grounded in unified real-time data through Data Cloud, they can connect IT issues directly to business impact, employee profiles, and customer revenue to proactively triage, route, and autonomously resolve issues.
Who is the CI Owner — how does ownership work in Agentforce ITSM?IT ServicePJF Notebook

In Agentforce ITSM, a Configuration Item is managed by a CI Owner — a user responsible for a specific asset or CI. They are assigned the "CI Owner" permission set group, granting access to create, own, and manage their assigned configuration items and related records.

The term "fulfiller" is reserved for personas who handle IT service workflows and tickets, not the underlying assets:

  • Incident Fulfiller: Triages and resolves incidents, can propose major incidents, and create related problems and change requests.
  • Problem Fulfiller: Investigates and manages the lifecycle of assigned problems, conducts root-cause analysis, implements solutions.
  • Change Fulfiller: Executes implementation of approved changes, develops change plans, coordinates task assignments, validates fixes through testing.
What certifications and compliance standards does ITSM meet?IT ServicePJF Notebook

Current Certifications

  • PinkVERIFY Certified: Confirms that service management workflows adhere to industry standards.
  • ITIL Compliant: End-to-end ITSM functionalities — incident, problem, and change management — are fully compliant with ITIL best practices.

Planned / Not Yet Available

  • FedRAMP and IL4 & 5: Not yet compliant with DoD public/private standards — on the product roadmap.
  • Dedicated IT Compliance Features: Specialized applications for "Regulations & Controls" and "Risk Mgmt. & Audit Compliance" not yet available — targeted for June 2026.
What framework can ITSM be packaged and distributed in?IT ServicePJF Notebook

1. Data Kits for Data Components

A Data Kit packages Data Cloud components — data model objects and data streams — so they can be readily deployed into an org. The predefined Agentforce IT Service Data Kit includes bundles of data streams (IT Service Core and Analytics bundles) and search indexes.

2. Salesforce Go "Solution Bundles"

Salesforce Go is a consolidated setup experience offering predefined Solution Bundles:

  • IT Service Standard Solution: Packages foundational processes — Incident, Problem, Change, Release Management, Self-Service Portal, SLA Management.
  • IT Service Enhanced Solution: Everything in Standard, plus CMDB and Agentforce capabilities.
  • IT Service Sample Data Solution: Packages sample data to instantly populate the org for exploration without manual data entry.

3. ITIL (Conceptual Framework)

The platform's workflows are packaged and distributed according to the ITIL framework — the industry-standard best practices that guide high-quality and efficient IT service management.
